###こんなんでどうでしょうか？

```
git checkout -b type/#issuesNumber/release_or_spike
```
- typeは以下から選択
   - fix(修正)
   - feature(機能追加)

- issueNumberはissueの連番を記述

- release_or_spike
   - release(upstreamBranch追加用)
   - spike(loacl作業用 ※ここはlocalなので、何でも良いかも？)

例）修正、issueNo.10でupstreamへのPRを投げる場合のbranch名

```
fix/#10/release
```
